# DriftIQ — Behavioral Intelligence Platform

> **"Know your behavioral baseline. Notice when it shifts."**

DriftIQ is a passive behavioral monitoring platform that builds a personal **Behavioral Digital Twin** from your daily device usage. It tracks when your patterns drift from your established baseline, assigns a behavioral risk level, and delivers AI-powered insights — all with zero invasive data collection.

## 🚀 Quick Start (One Command)

```bash
# Prerequisites: Docker + Docker Compose installed
git clone https://github.com/your-org/driftiq.git
cd driftiq
make up
```

This single command will:
1. Generate RSA keypair for JWT signing
2. Start PostgreSQL, Redis, Nginx
3. Build and launch the API server (port 8000)
4. Build and launch Celery worker + Beat scheduler
5. Run database migrations automatically

**API Documentation**: http://localhost:8000/docs  
**API Base URL**: http://localhost:8000/api/v1

## 🧠 How It Works

### 1. Data Collection
- Foreground `CollectionService` polls `UsageStatsManager` every 15 minutes
- Events stored in local Room database, synced to backend in batches

### 2. Behavioral Digital Twin
- **10 behavioral dimensions** tracked: sleep, screen time, unlocks, social, productivity, entertainment, learning, late-night usage, sessions, entropy
- Separate weekday/weekend **EWMA baselines** (α=0.1, slow adaptation)
- Baseline confidence grows over 30 days

### 3. Drift Detection (Spec §12)
- Per-dimension **z-scores** computed vs personalized baseline
- **Composite drift score** (0-100) with weighted average of z-scores
- **Drift velocity** from 3-day rolling slope

### 4. Risk Engine (Spec §13)
- **6 levels** (0=Healthy → 5=Critical) based on composite drift + velocity + sustained days
- All language is **behavioral-only** — no medical/clinical terms ever used

### 5. AI Insights
- GPT-4o generates 3-sentence daily behavioral observations
- Falls back to rule-based templates when no API key is configured
- Weekly and monthly summaries generated by Celery Beat

## 🔒 Privacy Principles

DriftIQ is **privacy-first by design**:

- ✅ Collects: app names, session durations, timestamps
- ❌ Never collects: message content, passwords, location, photos, keystrokes, microphone, financial data
- ✅ All data encrypted in transit (HTTPS)
- ✅ Data stored in your own database
- ✅ Full data export available
- ✅ Account deletion removes all data within 30 days

## 🚢 Production Deployment

1. Set all required environment variables (especially `SECRET_KEY` with a strong value)
2. Add `DEPLOY_HOST`, `DEPLOY_USER`, `DEPLOY_SSH_KEY` to GitHub Secrets
3. Push to `main` branch — CI/CD deploys automatically
4. For HTTPS: place the stack behind a TLS-terminating load balancer (e.g., Cloudflare, AWS ALB)
